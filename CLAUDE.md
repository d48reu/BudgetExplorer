# BudgetExplorer

Miami-Dade County's $13.2B FY 2025-26 budget as an interactive, searchable, plain-English visualization.

**Repo:** https://github.com/d48reu/BudgetExplorer.git
**Live:** Vercel (frontend) + Neon Serverless Postgres (database)
**Status:** v1.1 shipped 2026-03-01; full data-integrity audit fixed and deployed to production 2026-07-18 (corrected Appendix J capital attribution, search index budgets, 5-year history, regenerated AI descriptions)

## Architecture

Dual-stack project:
- `budget-explorer-web/` — Next.js 16 frontend (TypeScript, App Router, Prisma, Tailwind CSS 4)
- `pipeline/` — Python ETL pipeline (pdfplumber extracts budget PDFs → PostgreSQL)

## Tech Stack

**Frontend:** Next.js 16, React 19, TypeScript 5, Tailwind CSS 4, Prisma 7 (PrismaPg adapter), D3.js + Recharts, pnpm
**Pipeline:** Python 3, pdfplumber, psycopg2, Anthropic SDK (AI descriptions), Click CLI
**Database:** Neon Serverless PostgreSQL, Prisma ORM
**Testing:** Vitest

## Key Commands

```bash
# Frontend (from budget-explorer-web/)
pnpm dev          # Dev server on localhost:3000
pnpm build        # Production build (runs prisma generate first)
pnpm lint         # ESLint

# Pipeline (from root)
python -m pipeline migrate              # Apply pending SQL migrations
python -m pipeline run-all              # Extract → Load → Verify
python -m pipeline extract              # Extract PDF → JSON
python -m pipeline load                 # Load JSON → PostgreSQL
python -m pipeline verify               # Verify totals vs published figures
python -m pipeline seed-historical      # Seed 5 fiscal years of data
python -m pipeline.generate.descriptions   # Regenerate AI descriptions (needs ANTHROPIC_API_KEY)
python -m pipeline.load.seed_descriptions  # Seed reviewed descriptions JSON into DB
```

## Production Data Operations

- `DATABASE_URL` in `.env` points at **local Postgres**; production Neon is `DATABASE_URL_PROD` in `.env` (never committed).
- To run any pipeline command against production, export `DATABASE_URL` to the prod value first (`load_dotenv` does not override exported vars). Use the **direct** endpoint for migrations — strip `-pooler` from the Neon host.
- After any prod reseed: `REFRESH MATERIALIZED VIEW search_index`, then trigger a Vercel redeploy (pages are static with daily ISR, so stale HTML persists up to 24h otherwise).
- Seeders delete+insert inside a single transaction (commit on connection close), so live reads never see a half-empty table.

## Key Conventions

- All monetary values stored as **BigInt cents** — never floats
- Pages are statically generated with daily ISR (`revalidate = 86400`); only /search renders per-request
- Every chart has a data table fallback (accessibility)
- AI descriptions generated by pipeline, not at request time
- PDF is source of truth — pipeline verifies extracted totals against published figures
- `pipeline/migrations/` is the canonical schema; root `budget-explorer-schema.sql` is a retired pointer

## Data Model Invariants (learned the hard way — 2026-07 audit)

- **`department_budgets` is multi-row per (department, fiscal year):** one row per strategic area the department operates in, plus capital-only rows from Appendix J. Any "department total" MUST sum the slices — never read `department_budgets[0]`. `queries.ts` has `sumBudgetRows()` / `areaMembershipFilter()` for this.
- **Area membership of a budget row** = `COALESCE(db.strategic_area_id, d.strategic_area_id)` — matches `checker.py` and migration 003 views. Area pages must select rows by the viewed area, not by the department's home area.
- **`is_actual` semantics:** FY 2021-24 rows are actual spending (`true`); FY 2024-25+ are adopted budgets (`false`). Current-FY queries filter `is_actual = false`; the YoY history merges both, preferring adopted when a year has each.
- **Formatting:** `formatDollarsAbbreviated()` and friends take **cents** — never pre-multiply or pre-divide at call sites.
- **Appendix extractors are state machines** keyed on ALL-CAPS headers; unrecognized headers must warn and close the current block, never inherit the previous department (silent capital misattribution was the worst audit bug).

## Database Schema (14 models)

Core: `fiscal_years`, `strategic_areas`, `departments`, `department_budgets`, `department_expenditures`, `expenditure_categories`, `budget_descriptions`, `revenue_sources`, `revenue_by_source`, `millage_rates`, `capital_programs`, `glossary_terms`, `department_aliases`, `disbursements`
