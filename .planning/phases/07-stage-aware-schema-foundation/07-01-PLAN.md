Create `budget-explorer-web/scripts/` (does not exist) and write `snapshot.mjs` — zero dependencies, Node >= 18 built-ins only (global fetch, node:fs, node:path, node:crypto). Two subcommands:

1. `node scripts/snapshot.mjs capture --base URL --out LABEL`
   - Fetch `${base}/sitemap.xml`, extract URLs with a `loc` element regex (no XML library — see Don't Hand-Roll table in RESEARCH.md).
   - Rewrite each URL's origin to `--base` (sitemap embeds the canonical domain, which differs from localhost).
   - If `/search` is not in the sitemap, append `${base}/search` (the search shell must be covered per the locked decision).
   - Fetch every URL sequentially; any non-200 response: print URL + status, exit 1.
   - Save each response body to `.snapshots/LABEL/FILE.html` where FILE is the URL path with slashes replaced by underscores (root path = `_index`). Write `manifest.json` mapping url to { file, sha256, bytes }.
   - Print page count on success. Never snapshot sitemap.xml itself.

2. `node scripts/snapshot.mjs diff LABELA LABELB [--normalize]`
   - Exit 1 if the two labels' page sets differ (list missing/extra pages).
   - Tier 0 (default): compare raw SHA-256 per page. On mismatch, print the page name and the first ~20 differing lines (line-by-line report; the gate itself is hash equality).
   - Tier 1 (`--normalize`): before hashing, apply to BOTH sides: (a) empty out the body of every script element (keep the open/close tags so structural mismatches still surface), (b) replace every `/_next/static/...` asset path (chars up to the next quote/whitespace) with the placeholder `/_next/static/X`. Then compare as Tier 0. This strips exactly the build-artifact noise (buildId asset paths, hashed chunk names, RSC flight payload) while preserving all server-rendered user-visible HTML.
   - Exit 0 only if ALL pages match; print a per-page PASS/FAIL summary.

Wire `"snapshot": "node scripts/snapshot.mjs"` into budget-explorer-web/package.json scripts. Add `.snapshots/` to budget-explorer-web/.gitignore.
  </action>
