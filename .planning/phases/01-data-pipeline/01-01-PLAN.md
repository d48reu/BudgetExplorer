    IMPORTANT: The Budget in Brief PDF must be downloaded first. The pipeline must support both local path and URL download per user decision.

    1. Create `pipeline/extract/pdf_reader.py`:
       - download_pdf(url, output_path) function using requests library — downloads PDF from URL to local path if not already present
       - open_pdf(pdf_path_or_url) function — if input is URL, download first, then return pdfplumber.open() context
       - inspect_pdf(pdf_path) function — prints page count, dimensions, tables found per page, saves debug images to pipeline/data/debug/ directory using page.to_image().debug_tablefinder(). This is critical for tuning extraction.
       - MUST handle both local file path and URL per user locked decision

    2. Create `pipeline/extract/strategic_areas.py`:
       - extract_strategic_areas(pdf_path) -> list[dict] function
       - Each dict: { "name": str, "operating_budget": str, "capital_budget": str, "total_budget": str, "employee_count": str }
       - Use pdfplumber crop + table_settings tuned for the strategic area summary table
       - Page numbers and bbox coordinates MUST be determined by running inspect_pdf first — use constants at top of module (STRATEGIC_AREA_PAGES, STRATEGIC_AREA_BBOX)
       - Include debug_tablefinder() calls wrapped in a DEBUG flag for development
       - Handle multi-year columns if present (extract all fiscal year columns)

    3. Create `pipeline/extract/departments.py`:
       - extract_departments(pdf_path) -> list[dict] function
       - Each dict: { "strategic_area": str, "name": str, "operating_budget": str, "capital_budget": str, "total_budget": str, "employee_count": str }
       - Departments are grouped under strategic areas in the PDF — preserve this grouping
       - May span multiple pages — handle page iteration
       - Per-section table_settings (different sections may need different settings per RESEARCH.md Pitfall 1)

    4. Create `pipeline/extract/revenue.py`:
       - extract_revenue(pdf_path) -> list[dict] function
       - Each dict: { "source": str, "amount": str, "percentage": str }
       - Extract the 7 revenue source categories
       - Handle multi-year columns if present

    5. Create `pipeline/extract/expenditures.py`:
       - extract_expenditures(pdf_path) -> list[dict] function
       - Each dict: { "category": str, "amount": str, "percentage": str }
       - Extract expenditure category breakdown (salary, fringes, etc.)

    6. Create `pipeline/extract/millage.py`:
       - extract_millage(pdf_path) -> list[dict] function
       - Each dict: { "authority": str, "millage_rate": str, "is_county": bool }
       - Extract property tax millage rate table
       - Flag county vs non-county authorities (School Board, Children's Trust are not county)

    7. Create `pipeline/extract/penny.py`:
       - extract_penny(pdf_path) -> list[dict] function
       - Each dict: { "strategic_area": str, "cents_per_dollar": int }
       - Per user locked decision: extract and match the published penny/dollar breakdown exactly (do NOT calculate from totals)
       - If the data is in a graphic rather than a table, implement text extraction from the graphic region and parse the cent values

    8. Create `pipeline/extract/__init__.py`:
       - Import all extraction functions for convenient access
       - extract_all(pdf_path) -> dict function that calls all extractors and returns a combined dict with keys: strategic_areas, departments, revenue, expenditures, millage, penny
