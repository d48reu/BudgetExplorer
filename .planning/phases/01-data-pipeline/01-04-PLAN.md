  <action>
    Create pipeline/scripts/ directory if it does not exist. Create pipeline/scripts/generate_historical_csvs.py that:

    1. Reads extracted_data.json (default path: project root extracted_data.json, overridable via --input arg)
    2. Accesses data["appendix_c"]["departments"] — a list of 81 dicts, each with keys:
       - strategic_area, department, actual_21_22, actual_22_23, actual_23_24, budget_24_25, adopted_25_26, positions_25_26
    3. For each of the 4 prior fiscal years, generates a CSV file matching the template schema in pipeline/data/historical/template.csv:
       - fiscal_year: "FY 2021-22", "FY 2022-23", "FY 2023-24", "FY 2024-25"
       - strategic_area: from the appendix_c entry's strategic_area field
       - department_name: from the appendix_c entry's department field
       - department_alias: leave empty (the seed_historical.py resolver will match names)
       - operating_budget: value from the corresponding column (actual_21_22 etc.), converted from thousands string ("7,591") to whole dollars (7591000). Remove commas from the string, convert to int, multiply by 1000.
       - capital_budget: 0 (Appendix C only has operating data; capital is not available per prior year)
       - employee_count: leave empty (positions are only available for FY 2025-26)
       - is_actual: "true" for FY 2021-22, FY 2022-23, FY 2023-24 (these are labeled "Actual" in Appendix C); "false" for FY 2024-25 (labeled "Budget" in Appendix C — it is the adopted budget, not actuals)

    The column-to-fiscal-year mapping:
    - actual_21_22 → FY 2021-22 (is_actual=true)
    - actual_22_23 → FY 2022-23 (is_actual=true)
    - actual_23_24 → FY 2023-24 (is_actual=true)
    - budget_24_25 → FY 2024-25 (is_actual=false)

    Skip rows where the operating budget value is empty, None, or "0" for that fiscal year.

    Write CSV output using Python's csv.writer with quoting=csv.QUOTE_MINIMAL. Include the header row. Output directory defaults to pipeline/data/historical/ (overridable via --output-dir arg).

    Use click for the CLI interface. Make the script runnable as `python pipeline/scripts/generate_historical_csvs.py`.

    After generating files, print a summary: filename, row count, total operating budget for each file.

    1. Run `python -m pipeline.cli seed-historical --data-dir pipeline/data/historical/`
    2. This will invoke seed_all_historical() which:
       - Finds all fy_*_departments.csv files
       - Parses each via parse_historical_csv() (converts dollar amounts to cents)
       - Creates fiscal_year records for each prior year
       - Resolves department names using the 4-strategy cascade in resolve_department()
       - Seeds department_budgets rows for each resolved department
    3. After seeding, verify by querying the database:
       - `SELECT fy.label, COUNT(*) FROM department_budgets db JOIN fiscal_years fy ON fy.id = db.fiscal_year_id GROUP BY fy.label ORDER BY fy.label` should return 5 rows (one per fiscal year) each with 50+ department records
       - `SELECT DISTINCT fy.label FROM fiscal_years fy ORDER BY fy.label` should list all 5 fiscal years
    4. If any departments fail to resolve during seeding (logged as warnings), document them. These are expected for departments that existed historically but are not in the current schema — they will be logged but should not block the seeding process.
    5. Run `python -m pipeline.cli verify --fiscal-year "FY 2025-26"` to confirm the existing FY 2025-26 data is not affected by the historical seeding.

    NOTE: This task requires a running PostgreSQL database with DATABASE_URL set in .env.
