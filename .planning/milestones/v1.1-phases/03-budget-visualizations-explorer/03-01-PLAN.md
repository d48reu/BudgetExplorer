1. Create `src/components/charts/ChartContainer.tsx` -- a `'use client'` wrapper that:
   - Accepts `aspectRatio?: number`, `minHeight?: number` (default 300), `className?: string`, `children: (dimensions: { width: number; height: number }) => React.ReactNode` (render prop pattern).
   - Uses `useRef<HTMLDivElement>` and `ResizeObserver` in `useEffect` to measure container width.
   - Computes height from `aspectRatio` (if given) or `Math.max(minHeight, width * 0.6)`.
   - Renders `<Skeleton>` (from `@/components/ui/Skeleton`) while dimensions are null.
   - Once dimensions available, calls `children({ width, height })`.
   - Cleans up ResizeObserver on unmount.
   - Outer div: `className="w-full"` merged with passed className via `clsx`.

2. Create `src/components/charts/DataTableToggle.tsx` -- a `'use client'` generic component that:
   - Props: `chartLabel: string`, `data: T[]`, `columns: TableColumn<T>[]` (imported from `@/types/budget`), `children: React.ReactNode` (the chart), `defaultView?: 'chart' | 'table'` (default 'chart').
   - State: `showTable` boolean, toggled by button.
   - Toggle button: top-right, `text-sm text-mdc-blue hover:underline`, label switches between "View as table" and "View as chart". Add `aria-pressed={showTable}`.
   - When `showTable=true`: render a `<div role="region" aria-label="Data table: {chartLabel}">` containing a `<table>` with proper `<thead>/<tbody>`. Columns respect `align` prop ('right' for numbers). Use `col.format` if provided, otherwise `String(row[col.key])`.
   - When `showTable=false`: render children wrapped in a div.
   - For accessibility: ALSO render a visually hidden copy of the table (sr-only) when chart is shown, so screen readers always have tabular access. Use `className="sr-only"` on this copy.
   - Table styling: `w-full text-sm border-collapse`, header cells `py-2 px-3 font-medium text-text-secondary border-b border-border`, body cells `py-2 px-3 border-b border-border last:border-0`.

3. Create `src/components/charts/ChartTooltip.tsx` -- a `'use client'` component that:
   - Props: `content: React.ReactNode`, `children: React.ReactNode` (the trigger element), `open: boolean`, `onOpenChange: (open: boolean) => void`.
   - Uses `@floating-ui/react`: `useFloating` with `offset(8)`, `flip()`, `shift()`, `autoUpdate`. Uses `useRole(context, { role: 'tooltip' })`, `useDismiss`.

   - NOTE: This component is a controlled tooltip -- the parent chart manages open/close state (because chart elements manage their own hover/focus). The parent passes `open` and `onOpenChange` to control visibility, and wraps the hovered chart element as children.
