    1. Create `pipeline/transform/clean.py`:
       - dollars_to_cents(value: str) -> int function following RESEARCH.md Pattern 2 exactly:
         * Handle: $1,234,567 | 1,234,567 | ($1,234) | -$1,234 | 1234.56 | 0 | "" | "-" | "N/A" | None
         * Parentheses = negative (budget convention)
         * Remove all non-numeric chars except decimal point
         * Whole dollar amounts (no decimal) multiply by 100
         * NEVER use float for final value — round(float * 100) only when decimal present
         * Return 0 for empty/null/dash values
       - clean_percentage(value: str) -> float function:
         * Handle: "25.3%" | "25.3" | "" | None
         * Strip % sign, return float or None
       - clean_employee_count(value: str) -> int | None function:
         * Handle: "3,500" | "3500" | "" | None | "-"
         * Strip commas, return int or None
       - clean_department_name(name: str) -> str function:
         * Strip whitespace, normalize unicode quotes/dashes
         * Remove footnote markers (*, +, etc.)

    2. Create `pipeline/transform/validate.py`:
       - validate_extracted_data(data: dict) -> list[str] function:
         * Takes the output dict from extract_all()
         * Checks: strategic_areas is non-empty (expect 9), departments is non-empty (expect ~35), revenue is non-empty (expect 7), millage is non-empty
         * Checks: every department has a strategic_area field
         * Checks: every budget value is parseable by dollars_to_cents (not gibberish)
         * Returns list of warning/error messages (empty = valid)
       - validate_totals_rough(data: dict) -> list[str]:
         * Quick sanity check: sum of department operating budgets should be within 10% of $8.5B
         * Sum of strategic area budgets should roughly equal total
         * Returns warnings for anything suspicious

    3. Create `pipeline/transform/__init__.py`:

       - Test clean_percentage: "25.3%" -> 25.3, "" -> None

  <action>
    1. Create `pipeline/load/seed.py`:
       - All seed functions follow the idempotent delete-write pattern from RESEARCH.md Pattern 3:
         DELETE existing records for the target fiscal year, then INSERT new ones.

       - seed_fiscal_year(conn, label: str, start_date: str, end_date: str, totals: dict) -> int:
         * INSERT INTO fiscal_years (label, start_date, end_date, total_operating, total_capital, total_budget, total_employees, is_adopted)
         * totals dict has operating_cents, capital_cents, total_cents, employees
         * Use ON CONFLICT (label) DO UPDATE for idempotency
         * Return the fiscal_year_id

       - seed_department_budgets(conn, fiscal_year_id: int, departments: list[dict]):
         * DELETE FROM department_budgets WHERE fiscal_year_id = %s
         * For each department: look up department_id by matching name to departments table (use slug or ILIKE matching)
         * INSERT into department_budgets with operating_budget, capital_budget, total_budget (all in cents via dollars_to_cents), employee_count
         * Log warnings for departments that can't be matched (don't fail — log and continue)

       - seed_strategic_area_budgets(conn, fiscal_year_id: int, areas: list[dict]):
         * DELETE FROM strategic_area_budgets WHERE fiscal_year_id = %s
         * Look up strategic_area_id by name/slug
         * INSERT with operating_budget, capital_budget (cents), cents_per_dollar from penny data

       - seed_revenue(conn, fiscal_year_id: int, revenue: list[dict]):
         * DELETE FROM revenue_by_source WHERE fiscal_year_id = %s
         * Look up revenue_source_id by name
         * INSERT with amount (cents), percentage

       - seed_millage_rates(conn, fiscal_year_id: int, rates: list[dict]):
         * DELETE FROM millage_rates WHERE fiscal_year_id = %s
         * INSERT with authority, millage_rate (DECIMAL), is_county flag, display_order

       - seed_all(conn, fiscal_year_id: int, data: dict):
         * Orchestrates all seed functions in order: fiscal_year -> strategic_area_budgets -> department_budgets -> revenue -> millage
         * Wraps everything in a single transaction
         * data is the output from extract_all() with values already transformed by clean.py

    2. Update `pipeline/cli.py`:
       - Wire the `extract` command:
         * Download PDF if URL provided (via pdf_reader.download_pdf)
         * Call extract_all(pdf_path)
         * Call validate_extracted_data(data) — print warnings, halt on errors

         * Read extracted JSON from --data path

       - Wire the `run-all` command:
